# Metadata Release Under Differential Privacy Pipeline

This repository contains an experimental pipeline for releasing metadata under differential privacy over taxi GPS data. The code implements a noise-addition baseline plus six metadata selection layer that uses utility estimation and knapsack and Greedy optimization.

## What this code does

- Preprocesses taxi ping data into a 2D spatial grid.
- Applies a DP base mechanism to the clipped cell counts.
- Generates six candidate metadata fields (`tau_noise`, `tau_sens`, `tau_count`, `tau_group_count`, `tau_row_contrib`, `tau_agg_comp`).
- Computes per-field explanatory utility as posterior variance reduction.
- Compares three selection strategies:
  - Optimal knapsack
  - Greedy utility/cost ratio
  - Random baseline
- Produces experiment results and summary tables for publication.
- Generates plots for algorithm comparisons, privacy loss, and scalability.
- Includes a separate empirical privacy loss distribution analysis script.

## Main files

- `main_pipeline/dp_metadata_pipeline.py`
  - Core evaluation pipeline.
  - Loads `tax.csv`, preprocesses GPS points into 20 X 20 grid cells, and computes privacy-preserving metadata traces.
  - Calculates utility for each metadata field using posterior inference.
  - Runs knapsack optimization, greedy baseline, and random baseline.
  - Writes `experiment_results.csv`, `budget_sweep_results.csv`, and `scalability_results.csv`.
  - Dumps cell-level `U_i` / `\Gamma_i` records via `CellDumper`.

- `main_pipeline/plots.py`
  - Reads CSV output from `dp_metadata_pipeline.py`.
  - Produces PDF plots:
    - Algorithm showdown bar chart
    - Utility CDF plot
    - Privacy/utility budget trade-off
    - Privacy loss distribution
    - Scalability comparison
    - Uncertainty reduction visualization

- `main_pipeline/empirical_privacy_loss_dist.py`
  - Empirically evaluates privacy loss distributions for metadata fields.
  - Samples neighboring datasets and output draws to estimate log-likelihood ratios.
  - Saves histogram plots and a summary CSV.

- `main_pipeline/cell_dumper.py`
  - Helper for exporting per-cell region, budget, utilities, and metadata costs to JSON.
  - Used by `dp_metadata_pipeline.py` to produce `data/cells_real.json`.

## Dependencies

- Python 3.10+
- numpy
- pandas

- seaborn

Install dependencies with pip:

```bash
python -m pip install numpy pandas scipy matplotlib seaborn
```

## How to run

Before running the pipeline, download the Rome taxi dataset from CRAWDAD and save it as `main_pipeline/tax.csv`.

- CRAWDAD Rome Taxi Dataset: https://ieee-dataport.org/open-access/crawdad-romataxi

The code is written to execute from within `main_pipeline/`.

```bash
cd main_pipeline
python dp_metadata_pipeline.py
```

This will:

- load `tax.csv`
- run the experimental DP + metadata pipeline
- save `experiment_results.csv`
- save `budget_sweep_results.csv`
- save `scalability_results.csv`
- create `data/cells_real.json` under `main_pipeline/`

To generate plots, run:

```bash
cd main_pipeline
python -c "import plots; plots.plot_algorithm_showdown('experiment_results.csv')"
```

or call the desired plotting functions manually from `plots.py`.

To run the empirical privacy-loss analysis:

```bash
cd main_pipeline
python empirical_privacy_loss_dist.py
```

## Output files

- `experiment_results.csv` — region-level utility comparisons for knapsack, greedy, and random selectors.
- `budget_sweep_results.csv` — average utility over a range of privacy budgets.
- `scalability_results.csv` — runtime comparison across grid resolutions.
- `privacy_loss_distribution_summary.csv` — empirical privacy loss summary from `empirical_privacy_loss_dist.py`.
- PDF plots generated by `plots.py`.
- `data/cells_real.json` — exported cell-level metadata utility/cost records.
